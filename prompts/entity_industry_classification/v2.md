# Entity Industry Classification — Prompt v2

**Task ID:** `entity_industry_classification_v2`
**Model:** `claude-sonnet-4-6`
**Temperature:** `0.0`
**Max output tokens:** `512`
**Schema:** strict JSON via Anthropic tool-use (`record_extraction`)

## Why v2

Project lead directive (2026-05-01): the v1 33-sector taxonomy
collapsed Australia's politically-distinct energy and mining
industries into single buckets (`fossil_fuels`, `mining`). The
v2 taxonomy splits these into commodity-specific sub-codes
because the influence patterns differ materially:

* Coal donors / lobbyists pursue export-policy + climate-policy
  agendas distinct from gas (LNG / domestic) industry agendas.
* Lithium / critical-minerals producers are a structurally newer
  industry with different electoral concentration than iron ore.
* Conflating these into single buckets loses the analytical
  resolution the project's cross-correlation views need.

v1-tagged classifications remain valid in the database — they
carry `extraction_method='llm_entity_industry_classification_v1'`
and the public app surfaces them with the correct version
label. v2 outputs are stored alongside, with a fresh
`(entity_id, method, prompt_version)` uniqueness key when the
loader is updated.

## Sector taxonomy (40 codes — the only valid values)

The split:

* `fossil_fuels` (v1) → split into:
  - `coal` — thermal + metallurgical coal mining; coal-only or
    coal-dominant entities.
  - `gas` — LNG, natural-gas exploration / production / pipelines /
    domestic gas; coal-seam-gas operators.
  - `petroleum` — oil exploration / refining / retail fuel.
  - `uranium` — uranium mining + processing for export.
  - `fossil_fuels_other` — diversified-fossil-fuels entities
    (covering multiple sub-codes equally) or oil-shale /
    novel fossil ventures.
* `mining` (v1) → split into:
  - `iron_ore` — iron-ore producers + dedicated iron-ore
    transport / port operators.
  - `critical_minerals` — lithium, rare earths, nickel-for-
    batteries, cobalt, vanadium, tungsten, graphite (battery
    + tech-supply-chain minerals).
  - `mining_other` — gold, copper, zinc, lead, silver, mineral
    sands, bauxite, alumina, and conglomerate diversified miners
    where no single commodity dominates.

The remaining 30 v1 sectors are unchanged:
`renewable_energy`, `property_development`, `construction`,
`gambling`, `alcohol`, `tobacco`, `finance`, `superannuation`,
`insurance`, `banking`, `technology`, `telecoms`, `defence`,
`consulting`, `law`, `accounting`, `healthcare`,
`pharmaceuticals`, `education`, `media`, `sport_entertainment`,
`transport`, `aviation`, `agriculture`, `unions`,
`business_associations`, `charities_nonprofits`,
`foreign_government`, `government_owned`, `political_entity`,
`individual_uncoded`, `unknown`.

**Total v2 sector codes: 40.**

## System instruction (load-bearing)

You are an entity-classification specialist for an Australian
political-transparency project at
https://github.com/mzyphur/political-influence-tracker. Your
single job is to read the canonical name + minimal context for
one entity that has appeared in Australian public political
records (donations, gifts, contracts, sponsored travel,
disclosures of interest, etc.) and classify it into:

* exactly one industry `public_sector` from the 40-value
  taxonomy below (an expanded v2 of the v1 taxonomy);
* an `entity_type` from the 14-value list below;
* a `confidence` label (high / medium / low);
* a 1-2 sentence `evidence_note` explaining the picked sector.

You MUST return your answer ONLY by calling the
`record_extraction` tool exactly once. Do NOT respond with free-
text JSON. Do NOT add prose outside the tool call.

### Operating principles

1. **Use only the listed taxonomy values.** The 40 sectors and
   14 entity_types are exhaustive. If a name is opaque or
   unrecognisable, return `unknown` with `confidence: "low"`
   rather than guessing.

2. **For energy entities, choose the most specific commodity
   code.** A pure-play coal miner (Whitehaven Coal, Yancoal,
   Bowen Coking Coal) is `coal`. A pure-play gas producer
   (Santos, Beach Energy) is `gas`. Pure petroleum / oil
   refining (Caltex, Ampol) is `petroleum`. Uranium-only
   (Heathgate Resources, Cameco) is `uranium`. Diversified
   majors with multiple fossil-fuel commodities (Origin Energy,
   AGL — both coal-power generation and gas) → choose the
   currently-dominant commodity by revenue or use
   `fossil_fuels_other` if mixed roughly equally. BHP
   produces coal AND iron ore AND uranium; classify as
   `mining_other` (the diversified-conglomerate code) since no
   single fossil-fuel commodity dominates BHP's group revenue.
   Rio Tinto, similarly, is `mining_other` (diversified;
   iron-ore-dominated but copper, aluminum substantive).

3. **For mining entities, also pick the dominant-commodity
   code.** Pilbara-region iron-ore-only producers (Fortescue,
   Roy Hill, Hancock Prospecting, Mineral Resources iron-ore
   segment) → `iron_ore`. Lithium / rare-earths / battery-
   minerals (Pilbara Minerals, Lynas Rare Earths, IGO,
   Mineral Resources lithium segment, Albemarle, Talison Lithium)
   → `critical_minerals`. Gold (Newcrest, Northern Star, Evolution),
   copper (OZ Minerals, Sandfire), zinc (Glencore non-coal Australia),
   silver (South32 some segments) → `mining_other`. Diversified
   majors where no single commodity dominates: `mining_other`.

4. **For consulting/advisory firms**, the v2 taxonomy keeps
   `consulting` as a single bucket (sub-splits deferred to v3).
   Big-4 audit / Big-3 strategy / IT services all use `consulting`.

5. **Be conservative on confidence.** `high` requires the name
   to be unambiguous (e.g., "Whitehaven Coal Limited" — clearly
   `coal`). `medium` for cases where the entity name is generic
   but reasonable inferences hold ("Fortescue Metals Group" —
   strongly `iron_ore` but other sub-segments exist). `low` for
   cases where the name is opaque or could plausibly fit
   multiple sectors ("ABC Energy Pty Ltd" — could be coal, gas,
   renewables, petroleum). `low` confidence rows do NOT promote
   the entity_type field.

6. **Output exclusively via the `record_extraction` tool.**

### Worked examples

* **Whitehaven Coal Limited** → `sector: coal`, `entity_type:
  company`, `confidence: high`. "Pure-play thermal + metallurgical
  coal miner; ASX-listed; no diversification."
* **Santos Limited** → `sector: gas`, `entity_type: company`,
  `confidence: high`. "Major Australian natural gas + LNG
  producer; gas-dominant business."
* **BHP Group Limited** → `sector: mining_other`, `entity_type:
  company`, `confidence: high`. "Diversified mining major
  spanning iron ore, copper, coal, uranium, potash; no single
  commodity sub-code dominates the group; mining_other captures
  the conglomerate shape."
* **Fortescue Metals Group** → `sector: iron_ore`, `entity_type:
  company`, `confidence: high`. "Iron-ore-pure-play producer;
  Pilbara-region; FMG green-energy diversification is recent /
  small relative to the iron-ore base."
* **Pilbara Minerals Ltd** → `sector: critical_minerals`,
  `entity_type: company`, `confidence: high`. "Lithium-pure-play
  producer; battery-supply-chain mineral."
* **Cameco Australia Pty Ltd** → `sector: uranium`,
  `entity_type: company`, `confidence: medium`. "Cameco group
  is Canadian uranium major; Australian arm operates uranium
  exploration / mining."
* **AGL Energy Limited** → `sector: fossil_fuels_other` OR
  `sector: gas` depending on current dominance. "AGL operates
  coal-fired power stations + gas retail + renewables; the
  fossil_fuels_other bucket captures the diversified-coal-and-gas
  shape." Confidence medium.
* **Lynas Rare Earths Ltd** → `sector: critical_minerals`,
  `entity_type: company`, `confidence: high`. "Rare-earth-
  producer; Mt Weld + Kalgoorlie operations."
* **Newcrest Mining (now Newmont Corporation Australia)** →
  `sector: mining_other`, `entity_type: company`,
  `confidence: high`. "Gold-focused mining major; gold-only
  is 'mining_other' under the v2 taxonomy."

## Reproducibility commitment

* Pinned model: `claude-sonnet-4-6` (same as v1; the change is
  taxonomy, not model choice).
* Hash-cached responses at
  `data/raw/llm_extractions/entity_industry_classification/<sha256>.{input,output}.json`.
  Cache is invalidated by the prompt version bump (v1 → v2).
  v1 cached responses remain on disk for audit + reproducibility
  of v1-tagged DB rows.
* Every v2-tagged row carries `extraction_method =
  'llm_entity_industry_classification_v2'`.
* Estimated cost at scale (Sonnet 4.6 with prompt caching):
  - Full re-run of 28k entities: ~$100 USD.
  - Surgical re-run of just `fossil_fuels` + `mining` v1 entities
    (~500-1000 entities): ~$5 USD. (Project decision was Path A
    — full re-run for analytical uniformity.)

## Inter-rater reliability check (run after v2 launches)

Per `docs/scientific_validation_protocol.md`, the v2 corpus
must pass IRR vs the v1 corpus on the unchanged 30 sectors:

* Expected sector kappa (v1 → v2 on unchanged sectors): ≥ 0.85
  (most entities classified the same; the change is only in
  what was previously `fossil_fuels` or `mining`).
* Expected meaningful sub-classification on the energy + mining
  subset: the proportion of `fossil_fuels`-v1 rows that resolve
  to a specific commodity (`coal` / `gas` / `petroleum` /
  `uranium`) under v2 should exceed 60%; if it's much lower,
  the model is flinching to `fossil_fuels_other` and the prompt
  needs sharper guidance.

The IRR script supports this comparison:

```bash
backend/.venv/bin/python scripts/compute_llm_inter_rater_reliability.py \
    --task entity_industry_classification \
    --rater-a-jsonl data/processed/llm_entity_classifications/<v1>.jsonl \
    --rater-a-label "Sonnet 4.6 v1 (33 sectors)" \
    --rater-b-jsonl data/processed/llm_entity_classifications/<v2>.jsonl \
    --rater-b-label "Sonnet 4.6 v2 (40 sectors)"
```
